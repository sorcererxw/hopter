# Operations Docs

Use this directory for workflow, local development, validation, and agent
coordination details. For product or architecture decisions, start in
`docs/product/README.md` or `docs/planning/README.md`.

## Common Tasks

- Start or inspect the local dev loop: [`DEV_LOOP.md`](DEV_LOOP.md)
- Understand contributor workflow: [`CONTRIBUTING.md`](CONTRIBUTING.md)
- Run agent-team execution: [`AGENT_TEAM_WORKFLOW.md`](AGENT_TEAM_WORKFLOW.md)
- Work on Codex app-server integration: [`CODEX_APP_SERVER_DEVELOPMENT_CONSTRAINTS.md`](CODEX_APP_SERVER_DEVELOPMENT_CONSTRAINTS.md)
- Record runtime artifacts and validation evidence: [`RUNTIME_ARTIFACTS_AND_VALIDATION.md`](RUNTIME_ARTIFACTS_AND_VALIDATION.md)
- Apply UI implementation rules: [`UI_SYSTEM_RULES.md`](UI_SYSTEM_RULES.md)

## Validation

The repo requires evidence before claiming completion. Use
`docs/VALIDATION_HARNESS.md` for the validation map and record evidence under
`.omx/evidence/` from the workspace root.
