## Important architectural decisions preserved

- Codex remains the source of truth for session content/history/approval semantics.
- `orchd` stores only lightweight session references and validation evidence.
- Browser never talks to Codex directly.
- Session detail hierarchy remains:
  1. status
  2. summary
  3. attention
  4. artifacts
  5. timeline
  6. terminal

## Known limitations / follow-up work

These are not blockers for the current validated alpha baseline, but are the most obvious next improvements:

1. WebSocket layer currently triggers refetch; it is not yet doing fine-grained cache patching.
2. Artifact viewers are still basic:
   - text works
   - richer diff/image viewers should be improved
3. Terminal drawer is still placeholder UX:
   - PTY-quality resize/streaming still needs a proper implementation
4. Auth/security can be hardened further for broader production use.
5. CI should automate the validation scripts and evidence publishing.
6. The working tree should be reviewed and committed in intentional chunks.
