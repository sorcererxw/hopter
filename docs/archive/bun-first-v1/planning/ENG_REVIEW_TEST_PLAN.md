# Test Plan

Generated by autoplan review

Scope: Codex-first self-hosted gateway server plus browser control plane

## Affected Pages / Routes

- `/` or dashboard
  - Verify active sessions, pending approvals, and degraded host states are visible without ambiguity.

- `/projects/:projectId`
  - Verify project selection, recent sessions, and start-session flow.

- `/sessions/:sessionId`
  - Verify plan display, checkpoint timeline, artifacts, and resume/attach behavior.

- `/settings/host`
  - Verify backend detection, host health, repo allowlist, and access mode configuration.

- `/settings/access`
  - Verify local-only mode, self-managed remote mode, and relay-mode configuration states.

## Key Interactions to Verify

- Start first session from an existing local repo bound to Codex.
- Approve a pending gate from a second device.
- Send a short steer/clarification reply to a running session.
- Interrupt, stop, and resume a session cleanly.
- Reconnect browser after network interruption and recover session state.
- Render artifacts:
  - summary
  - test output
  - screenshot
  - changed-files list

## Edge Cases

- Codex installed but incompatible version
- Codex unavailable after a host restart
- Gateway restart during a running session
- Browser disconnect during pending approval
- Artifact exists in timeline but cannot be fetched
- Relay unavailable while host remains healthy
- Session can be read from history but cannot be reattached live
- Multiple devices open the same session detail page

## Critical Paths

- Install gateway on home machine -> detect Codex -> add repo -> start session remotely
- Leave home machine -> open second device -> inspect current session -> approve next step
- Gateway/browser disconnect -> reconnect -> recover current session timeline without data loss
- Running session produces artifacts -> user inspects them remotely -> decides whether to continue

## Required Test Types

- Unit
  - gateway adapter event translation
  - degraded-state mapping
  - artifact metadata normalization

- Integration
  - gateway <-> Codex app-server handshake
  - session create/resume/interrupt paths
  - approval submit/retry logic

- E2E
  - first remote session from another device
  - reconnect flow after browser disconnect
  - pending approval flow on mobile viewport

## Critical Gaps To Cover First

- auth and trust-boundary tests for local-only vs remote exposure modes
- reconnect/source-of-truth tests after gateway restart or partial event loss
- compatibility tests for Codex version skew and backend capability changes
